Demoscript als Download

Hier mchten wir euch mit einer Sammlung von Tools vertraut machen, die spezifisch fr das Daten prozessieren in Data Science entwickelt wurden. Der Prozess und das Modell ist hier1 schn beschrieben. Die Sammlung von Tools wird unter dem Namen tidyverse vertrieben, welches wir ja schon zu Beginn der ersten bung installiert und geladen haben. Die Tools erleichtern den Umgang mit Daten ungeheuer und haben sich mittlerweile zu einem must have im Umgang mit Daten in R entwickelt.

Wir knnen Euch nicht smtliche Mglichkeiten von tidyverse zeigen. Wir fokussieren uns deshalb auf einzelne Komponenten2 und zeigen ein paar Funktionalitten, die wir oft verwenden und Euch ggf. noch nicht bekannt sind. Wer sich vertieft mit dem Thema auseinandersetzen mchte, der sollte sich unbedingt das Buch @wickham2017 beschaffen. Eine umfangreiche, aber nicht ganz vollstndige Version gibt es online3, das vollstndige eBook kann ber die Bibliothek bezogen werden4.

Packete laden

library(tidyverse)

Mit library(tidyverse) werden nicht alle Packete geladen, die mit install.packages(tidyverse) intalliert wurden (warum?). Unter anderem muss lubridate noch separat geladen werden:

library(lubridate)

wetter <- read_table("order_52252_data.txt",

                    col_double()
                    )

Kennwerte berechnen

Wir mchten den Mittelwert aller gemessenen Temperaturwerte berechnen. Dazu knnten wir folgenden Befehl verwenden:

mean(wetter$tre200h0, na.rm = TRUE) 

[1] 8.962106

Die Option na.rm = T bedeutet, dass NA Werte von der Berechnung ausgeschlossen werden sollen.

Mit der selben Herangehensweise knnen diverse Werte berechnet werden (z.B. das Maximum (max()), Minimum (min()), Median (median()) u.v.m.).

Diese Herangehensweise funktioniert nur dann gut, wenn wir die Kennwerte ber alle Beobachtungen (Zeilen) fr eine Variable (Spalte) berechnen wollen. Sobald wir die Beobachtungen gruppieren wollen, wird es schwierig. Zum Beispiel, wenn wir die durchschnittliche Temperatur pro Jahr berechnen wollen.

Convenience Variablen

Um diese Aufgabe zu lsen, muss zuerst das Jahr berechne werden (das Jahr ist die convenience variabel). Hierfr brauchen wir die Funktion year() (von lubridate).

Nun kann kann die convenience Variable Jahr erstellt werden. Ohne dpylr wird eine neue Spalte wird folgendermassen hinzugefgt. 
wetter$year <- year(wetter$time)
Mit dplyr (siehe 5) sieht der gleiche Befehl folgendermassen aus: 
wetter <- mutate(wetter,year = year(time))

Der grosse Vorteil von dplyr ist an dieser Stelle noch nicht ersichtlich. Dieser wird aber spter klar.

Kennwerte nach Gruppen berechnen

Jetzt kann man die data.frame mithilfe der Spalte Jahr filtern. 
mean(wetter$tre200h0[wetter$year == 2000], na.rm = TRUE)

[1] 9.281542

Dies mssen wir pro Jahr wiederholen, was natrlich sehr umstndlich ist, v.a. wenn man eine Vielzahl an Gruppen hat (z.B. Kalenderwochen statt Jahre). Deshalb nutzen wir das package dplyr. Damit geht die Aufgabe (Temperaturmittel pro Jahr berechnen) folgendermassen:

summarise(group_by(wetter,year),temp_mittel = mean(tre200h0, na.rm = TRUE))

Verketten vs.verschachteln

Auf Deutsch bersetzt heisst die obige Operation folgendermassen:

  1. nimm den Datensatz wetter
  2. Bilde Gruppen pro Jahr (group_by(wetter,year))
  3. Berechne das Temperaturmittel (mean(tre200h0))
Diese bersetzung R-> Deutsch unterscheidet sich vor allem darin, dass die Operation auf Deutsch verkettet ausgesprochen wird (Operation 1->2->3) whrend der Computer verschachtelt liest 3(2(1)). Um R nher an die gesprochene Sprache zu bringen, kann man den %>%-Operator verwenden (siehe 6). 
summarise(group_by(wetter,year),temp_mittel = mean(tre200h0))

# wird zu:

wetter %>%                                #1) nimm den Datensatz "wetter"
  group_by(year) %>%                      #2) Bilde Gruppen pro Jahr
  summarise(temp_mittel = mean(tre200h0)) #3) berechne das Temperaturmittel

Dieses Verketten mittels %>% macht den Code einiges schreib- und leserfreundlicher, und wir werden ihn in den nachfolgenden bungen verwenden. Dabei handelt es sich um das package magrittr, welches mit tidyverse mitgeliefert wird.

Zu dplyr und magrittrgibt es etliche Tutorials online (siehe7), deshalb werden wir diese Tools nicht in allen Details erlutern. Nur noch folgenden wichtigen Unterschied zu zwei wichtigen Funktionen in dpylr: mutate() und summarise().

  • summarise() fasst einen Datensatz zusammen. Dabei reduziert sich die Anzahl Beobachtungen (Zeilen) auf die Anzahl Gruppen (z.B. eine zusammengefasste Beobachtung (Zeile) pro Jahr). Zudem reduziert sich die Anzahl Variablen (Spalten) auf diejenigen, die in der summarise Funktion spezifiziert wurde (z.B. temp_mittel).
  • mit mutate wird ein data.frame vom Umfang her belassen, es werden lediglich zustzliche Variablen (Spalten) hinzugefgt (siehe Beispiel unten).
# Maximal und minimal Temperatur pro Kalenderwoche
wetter %>%                              #1) nimm den Datensatz "wetter"
  filter(stn == "ABO") %>%              #2) filter auf Station namnes "ABO"
  mutate(kw = week(time)) %>%       #3) erstelle eine neue Spalte "kw"
  group_by(kw) %>%                      #4) Nutze die neue Spalte um Guppen zu bilden
  summarise(
    temp_max = max(tre200h0, na.rm = TRUE),#5) Berechne das Maximum 
    temp_min = min(tre200h0, na.rm = TRUE) #6) Berechne das Minimum
    )

Resultate plotten

Mit diesen Tools knnen wir nun auch eine neue Grafik plotten, hnlich wie in der bung 1. Dafr mssen wir die ganzen Operationen aber zuerst in einer Variabel speichern (bis jetzt hat R zwar alles schn berechnet, aber uns nur auf die Konsole ausgegeben).

wetter_sry <- wetter %>%                              
  mutate(
    kw = week(time)
    ) %>%
  filter(stn == "ABO") %>%
  group_by(kw) %>%                      
  summarise(
    temp_max = max(tre200h0),               
    temp_min = min(tre200h0),
    temp_mean = mean(tre200h0)
    )

Dieses Mal plotten wir nur mit ggplot2 (siehe 8)

ggplot() +
  geom_line(data = wetter_sry, aes(kw,temp_max), colour = "yellow") +
  geom_line(data = wetter_sry, aes(kw,temp_mean), colour = "pink") +
  geom_line(data = wetter_sry, aes(kw,temp_min), colour = "black") +
  labs(y = "temp")

Das sieht schon mal gut aus. Nur, wir mussten pro Linie einen eigene Zeile schreiben (geom_line()) und dieser eine Farbe zuweisen. Bei drei Werten ist das ja ok, aber wie sieht es denn aus wenn es Hunderte sind? Da hat ggplot natrlich eine Lsung, dafr mssen aber alle Werte in einer Spalte daher kommen. Das ist ein hufiges Problem: Wir haben eine breite Tabelle (viele Spalten), bruchten aber eine lange Tabelle (viele Zeilen).

Reshaping data

Breit -> lang

Da kommt tidyverse wieder ins Spiel. Die Umformung von Tabellen breit->lang erfolgt mittels tidyr(siehe 9). Auch dieses package funktioniert wunderbar mit piping (%>%).

  pivot_longer(c(temp_max,temp_min,temp_mean))

Wenn wir die Namen neuen Spalten festlegen wollen (anstelle von name und value) erreichen wir dies mit names_to bzw. values_to:

wetter_sry_long <- wetter_sry %>%
  pivot_longer(-kw, names_to = "Messtyp", values_to = "Messwert")

Lang -> breit

Das Gegenstck zu pivot_longer ist pivot_wider. Mit dieser Funktion knnen wir eine lange Tabelle in eine breite berfhren. Dazu mssen wir in names_from angeben, aus welcher Spalte die neuen Spaltennamen erstellt werden sollen (names_from) und aus welcher Spalte die Werte entstammen sollen (values_from):

wetter_sry_long %>%
  pivot_wider(names_from = Messtyp, values_from = Messwert)
