Kennwerte berechnen

Wir mÃ¶chten den Mittelwert aller gemessenen Temperaturwerte berechnen. Dazu kÃ¶nnten wir folgenden Befehl verwenden:

[1] 6.324744

Die Option na.rm = T bedeutet, dass NA Werte von der Berechnung ausgeschlossen werden sollen.

Mit der selben Herangehensweise kÃ¶nnen diverse Werte berechnet werden (z.B. das Maximum (max()), Minimum (min()), Median (median()) u.v.m.).

Diese Herangehensweise funktioniert nur dann gut, wenn wir die Kennwerte Ã¼ber alle Beobachtungen fÃ¼r eine Variable (Spalte) berechnen wollen. Sobald wir die Beobachtungen gruppieren wollen, wird es schwierig. Zum Beispiel, wenn wir die durchschnittliche Temperatur pro Monat berechnen wollen.

Convenience Variablen

Um diese Aufgabe zu lÃ¶sen, muss zuerst den Monat extrahiert werden (der Monat ist die convenience variabel). HierfÃ¼r brauchen wir die Funktion lubridate::month().

Nun kann kann die convenience Variable â€œMonthâ€ erstellt werden. Ohne dpylr wird eine neue Spalte folgendermassen hinzugefÃ¼gt.

Mit dplyr (siehe)3 sieht der gleiche Befehl folgendermassen aus:

Der grosse Vorteil von dplyr ist an dieser Stelle noch nicht ersichtlich. Dieser wird aber spÃ¤ter klar.

Verketten vs.Â verschachteln

Auf Deutsch Ã¼bersetzt heisst die obige Operation folgendermassen:

  1. nimm den Datensatz wetter
  2. Bilde Gruppen pro Jahr (group_by(wetter,year))
  3. Berechne das Temperaturmittel (mean(tre200h0))

Diese Ãœbersetzung R-> Deutsch unterscheidet sich vor allem darin, dass die Operation auf Deutsch verkettet ausgesprochen wird (Operation 1->2->3) wÃ¤hrend der Computer verschachtelt liest 3(2(1)). Um R nÃ¤her an die gesprochene Sprache zu bringen, kann man den %>%-Operator verwenden (siehe).4

# 1 nimm den Datensatz "wetter"
# 2 Bilde Gruppen pro Monat
# 3 berechne das Temperaturmittel 

summarise(group_by(wetter,month),temp_mittel = mean(tre200h0))
#                  \_1_/
#         \__________2_________/
#\___________________3_______________________________________/

# wird zu:

wetter %>%                                 # 1
  group_by(month) %>%                      # 2
  summarise(temp_mittel = mean(tre200h0))  # 3

Dieses Verketten mittels %>% (genannt â€œpipeâ€) macht den Code einiges schreib- und leserfreundlicher, und wir werden ihn in den nachfolgenden Ãœbungen verwenden. Die â€œpipeâ€ wird mit dem package magrittr bereitgestellt und mit dplyr mitinstalliert.

Zu dplyr gibt es etliche Tutorials online (siehe5), deshalb werden wir diese Tools nicht in allen Details erlÃ¤utern. Nur noch folgenden wichtigen Unterschied zu zwei wichtigen Funktionen in dpylr: mutate() und summarise().

  • summarise() fasst einen Datensatz zusammen. Dabei reduziert sich die Anzahl Beobachtungen (Zeilen) auf die Anzahl Gruppen (z.B. eine zusammengefasste Beobachtung (Zeile) pro Jahr). Zudem reduziert sich die Anzahl Variablen (Spalten) auf diejenigen, die in der â€œsummariseâ€ Funktion spezifiziert wurde (z.B. temp_mittel).
  • mit mutate wird ein data.frame vom Umfang her belassen, es werden lediglich zusÃ¤tzliche Variablen (Spalten) hinzugefÃ¼gt (siehe Beispiel unten).
